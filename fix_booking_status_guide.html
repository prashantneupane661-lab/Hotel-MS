Fix Booking Status Workflow

Simple 2-step fix to show "Pending" status correctly

Current Issue:

When customers book a room, the Status column shows as empty instead of Pending

Step 1 Fix Database Column

This ensures the status column has proper default value and ENUM options

📋 Instructions:
  1. Open phpMyAdmin: http://localhost/phpmyadmin
  2. Click on database: hotel_management
  3. Click SQL tab at the top
  4. Copy and paste the code below
  5. Click "Go" button
-- Fix the bookings status column ALTER TABLE bookings MODIFY COLUMN status ENUM('pending','confirmed','rejected','checked_in','checked_out','cancelled') NOT NULL DEFAULT 'pending'; -- Fix any existing empty statuses UPDATE bookings SET status = 'pending' WHERE status IS NULL OR status = ''; -- Verify the fix SELECT status, COUNT(*) as total FROM bookings GROUP BY status;
✅ Expected Result: Query executed successfully. All bookings now have status = 'pending' or valid value.

Step 2 Test the Complete Workflow

  3. Go to: All Bookings

  6. Click
